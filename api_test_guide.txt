========================================================================
[Boasse Backend] API 테스트 가이드 (최종: 비밀번호 인증 방식)
========================================================================

1. 공지사항 목록 조회 (GET)
------------------------------------------------------------------------
- URL: http://localhost:8080/api/v1/notices?page=1&limit=5
- 설명: 누구나 조회 가능하며, 1페이지에 5개씩 최신순으로 가져옵니다.

2. 공지사항 작성 (POST)
------------------------------------------------------------------------
- URL: http://localhost:8080/api/v1/notices
- Body (form-data):
    - title: [긴급] 2026학년도 수강신청 일정 안내
    - content: 수강신청은 1월 20일 10시부터 시작됩니다. 자세한 내용은 첨부파일을 확인하세요.
    - password: admin1234
    - files: (File 선택) notice_guide.pdf
- 설명: 관리자 비밀번호(admin1234)가 맞아야 등록됩니다.

3. 공지사항 상세 조회 (GET)
------------------------------------------------------------------------
- URL: http://localhost:8080/api/v1/notices/1
- 설명: 게시글 ID가 1인 공지사항의 상세 내용과 첨부파일 목록을 가져옵니다.

4. 공지사항 수정 (PUT)
------------------------------------------------------------------------
- URL: http://localhost:8080/api/v1/notices/1
- Body (form-data):
    - title: (수정) 2026학년도 수강신청 일정 및 장소 안내
    - content: 장소 정보가 추가되었습니다. 학생회관 2층입니다.
    - password: admin1234
    - files: (File 선택) map_info.png
    - removeFileIds: 1
- 설명: 기존 파일(ID: 1)을 삭제하고 새 파일을 추가하며 제목/내용을 수정합니다. 
  비밀번호가 일치해야 합니다.

5. 공지사항 삭제 (DELETE)
------------------------------------------------------------------------
- URL: http://localhost:8080/api/v1/notices/1?password=admin1234
- 설명: 게시글 ID 1번을 삭제합니다. 비밀번호는 URL 파라미터로 전달합니다.

========================================================================
* 에러 테스트 (403 Forbidden):
- 위 작업 시 password에 "1234" 등 틀린 값을 입력하면 "관리자 비밀번호가 일치하지 않습니다." 에러가 발생합니다.
